Fraud detection starts with exploration. You need to inspect class imbalance, transaction distributions, feature patterns, and how sampling methods change the learning signal before judging model quality.

The project includes a generated dataset of 6,000 transactions, 117 fraud cases, and 8 numeric features. You can also swap in a different CSV for future experiments.

EDA Highlights

Check the transaction amount distribution, time-based variation, and feature skewness. In this dataset, the amount feature emerged as the strongest predictor, followed by f6, f4, f1, and f3.

Sampling Tradeoffs

Undersampling improves balance quickly and pushed recall to 0.9655 in this run, though precision dropped sharply. SMOTE produced a more balanced middle ground, while the baseline model kept the best precision.

Decision Guidance

Use a lower threshold when recall matters most and missed fraud is expensive. Use a higher threshold when review resources are tight and customer experience must remain smooth. The current trained baseline uses a threshold of 0.3214.
